## 🚀 运行方式

### 方式1: Web 版本（推荐测试）

```bash
cd w:\work_dir\breakfast_checkin_app

# 配置环境变量
$env:PUB_HOSTED_URL = "https://pub.flutter-io.cn"
$env:FLUTTER_STORAGE_BASE_URL = "https://storage.flutter-io.cn"
$env:PATH = "w:\work_dir\flutter\bin;$env:PATH"

# 获取依赖
flutter pub get

# 运行Web版本
flutter run -d chrome
```

### 方式2: Windows 桌面版

```bash
# 需要开启开发者模式
# 设置 -> 隐私和安全性 -> 开发者模式 -> 开启

flutter run -d windows
```

## 🧪 测试用例

### 测试1: 打卡功能

**步骤:**
1. 打开首页，查看中央胆结石状态
2. 点击橙色"打卡"按钮
3. 观察按钮变为绿色"已打卡"
4. 查看底部提示文字变为"今日已打卡 XX:XX"

**预期结果:**
- ✅ 打卡按钮变为绿色，显示"已打卡"
- ✅ 显示打卡成功提示
- ✅ 连续打卡天数增加
- ✅ 胆结石状态更新（如果之前有漏打卡）

---

### 测试2: 胆结石视觉效果

**步骤:**
1. 首页查看中央圆形结石显示
2. 记录当前结石大小和颜色
3. 切换到底部"健康"标签
4. 查看大图展示

**预期结果:**
- ✅ 0天漏打卡: 绿色 ✓ 图标
- ✅ 1天漏打卡: 黄色小结石 (10%)
- ✅ 2天漏打卡: 橙色结石 (25%)
- ✅ 3天漏打卡: 橙红色结石 (45%)
- ✅ 4天漏打卡: 红色大结石 (65%)
- ✅ 5天+漏打卡: 棕色巨大结石 (85%)

---

### 测试3: 统计页面

**步骤:**
1. 点击底部"统计"标签
2. 查看顶部三个数据卡片
3. 向下滚动查看日历热力图
4. 继续向下查看趋势图

**预期结果:**
- ✅ 显示"连续打卡"天数
- ✅ 显示"本月达标"百分比
- ✅ 显示"总打卡"次数
- ✅ 日历显示已打卡日期（绿色✓标记）
- ✅ 今天显示橙色边框
- ✅ 趋势图显示近7天打卡情况

---

### 测试4: 提醒设置

**步骤:**
1. 点击底部"提醒"标签
2. 点击右上角"+"按钮
3. 选择时间（如 08:00）
4. 保存提醒
5. 点击开关测试启用/禁用
6. 长按提醒项删除

**预期结果:**
- ✅ 成功添加提醒，显示在列表中
- ✅ 时间显示为 HH:MM 格式
- ✅ 开关可控制启用/禁用
- ✅ 禁用后提醒不再触发
- ✅ 长按可删除提醒

**注意:** Web版本不支持本地通知，只能在移动设备或桌面应用测试通知功能。

---

### 测试5: 数据持久化

**步骤:**
1. 在首页打卡
2. 关闭应用（完全退出）
3. 重新打开应用

**预期结果:**
- ✅ 重新打开后仍显示"已打卡"状态
- ✅ 连续打卡天数保留
- ✅ 历史记录保留

## ⚠️ 已知限制

1. **Web版本限制:**
   - 本地通知功能不可用（需要移动/桌面平台）
   - 数据库使用内存模式（刷新页面数据丢失）

2. **Windows版本:**
   - 需要开启开发者模式才能构建
   - 首次运行需要下载依赖

3. **美术资源:**
   - 当前使用占位实现（渐变色圆形）
   - 设计图到位后可无缝替换
